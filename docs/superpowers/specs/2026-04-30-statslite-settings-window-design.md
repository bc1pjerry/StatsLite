# StatsLite 设置窗口设计

日期：2026-04-30

## 目标

为 StatsLite 增加一个独立设置窗口，用于配置刷新间隔、菜单栏颜色和显示指标，并在窗口中展示相关系统指标。界面采用深色仪表风格，让应用从单纯菜单栏读数扩展为轻量监控控制台。

## 入口

设置窗口采用 macOS 原生 `Settings` 场景承载，并从菜单栏下拉菜单打开：

- 菜单栏菜单新增 `Settings...` 项，位于指标信息和退出按钮之间。
- 点击 `Settings...` 打开或聚焦设置窗口。
- 应用仍作为 accessory 菜单栏工具运行，不显示 Dock 图标，不在启动时显示主窗口。

## 视觉方向

界面采用方案 B：深色仪表风。

设计特征：

- 主背景使用深石墨色。
- 设置区和指标卡使用略浅的深色面板。
- 分割线、边框和辅助文字保持低对比，避免界面显得厚重。
- 强调色使用高可读的青绿色作为默认色，并提供蓝色、暖橙等预设。
- 指标卡以大号百分比或设备名作为核心读数，标签保持小而清晰。

界面应保持现代感，但仍遵循 macOS 设置窗口的约束：结构清晰、控件可预测、密度适中。

## 窗口结构

设置窗口使用两栏布局：

- 左侧窄侧边栏：`常规`、`外观`、`指标`。
- 右侧内容区：根据所选侧边栏项显示设置和指标。

第一版使用一个 SwiftUI 设置窗口完成，不创建额外弹出面板。侧边栏负责切换分组，右侧根据当前分组显示对应设置，避免把所有控件堆在单页中。

## 常规

常规页负责刷新行为：

- 提供刷新间隔选择：`1s`、`2s`、`5s`、`10s`。
- 默认值保持当前行为，为 `2s`。
- 切换刷新间隔后，菜单栏刷新定时器即时重启并使用新间隔。
- 菜单栏菜单中的 `Refresh: Ns` 使用同一份设置值。

刷新间隔配置需要持久化到 `UserDefaults`，应用重启后保持用户选择。

## 外观

外观页负责菜单栏仪表颜色：

- 提供预设强调色：青绿、蓝色、暖橙。
- 默认值为当前低饱和青绿色。
- 设置窗口中的指标卡和菜单栏半圆进度条使用同一强调色。
- 背景轨道继续使用系统灰或深色模式下合适的低对比灰。

第一版只支持预设色，不实现任意颜色选择器，避免扩大实现范围。

## 指标

指标页负责菜单栏显示内容和当前系统状态：

- 显示 CPU、内存和 GPU 相关指标。
- CPU 显示当前使用率整数百分比。
- 内存显示当前使用率整数百分比，并保留已用/总量摘要。
- GPU 显示当前默认 GPU 名称。
- 提供菜单栏显示组合选择，默认保持当前的 `CPU + Memory`。

第一版菜单栏显示组合支持：

- `CPU + Memory`
- `CPU only`
- `Memory only`

GPU 先作为设置窗口和菜单下拉菜单里的信息展示，不作为菜单栏半圆仪表显示项。GPU 当前只有设备名，没有稳定的使用率数据，直接放入菜单栏仪表会让显示语义不一致。

## 设置模型

新增一个轻量偏好模型，负责读写和发布设置变更：

- 刷新间隔。
- 强调色预设。
- 菜单栏指标组合。

偏好模型使用 `UserDefaults` 持久化，提供默认值，并暴露 SwiftUI 可观察状态。`MenuBarController` 监听设置变化并即时更新：

- 重启刷新定时器。
- 重绘菜单栏仪表。
- 重建菜单内容。

## 数据流

系统指标采集继续由 `SystemStatsProvider` 负责。

数据流为：

1. `SystemStatsProvider` 生成 `StatsSnapshot`。
2. `MenuBarController` 按设置中的刷新间隔定时读取快照。
3. `MenuBarController` 根据指标组合和强调色更新菜单栏视图。
4. 设置窗口读取同一份偏好模型，并展示最新设置。
5. 设置变化写入 `UserDefaults`，同时通知菜单栏控制器刷新。

设置窗口中的指标读取共享的最新快照状态。`MenuBarController` 在每次刷新时发布 `latestSnapshot`，设置窗口订阅同一份状态，不独立创建第二个定时器或第二个 `SystemStatsProvider`，避免重复采样和状态分叉。

## 错误与边界

- 刷新间隔只接受预设值；若 `UserDefaults` 中存在非法值，回退到 `2s`。
- 强调色若无法解析，回退到默认青绿色。
- 指标组合若无法解析，回退到 `CPU + Memory`。
- 若 GPU 不可用，继续显示 `Unavailable`。
- 若内存总量为 0，内存使用率显示为 0%，延续当前格式化行为。

## 测试

新增或扩展单元测试覆盖确定性逻辑：

- 设置默认值正确。
- 刷新间隔非法值会回退到默认值。
- 强调色预设可以稳定映射到显示颜色。
- 菜单栏指标组合默认值和非法值回退正确。
- 菜单栏布局可以根据 `CPU + Memory`、`CPU only`、`Memory only` 计算稳定尺寸。

实现完成后运行：

- `xcodebuild test`
- 必要时运行 App 做冒烟验证，确认菜单栏显示、设置窗口打开、设置即时生效。

## 第一版不包含

- 菜单栏弹出式完整状态面板。
- 历史曲线和趋势图。
- GPU 使用率仪表。
- 任意颜色选择器。
- 登录时自动启动。
- 多套主题或自动跟随系统深浅色。
